## 第二阶段：特征工程适配模块开发

### 2.1 实时特征生成器

**AI提示词**：
```
请帮我开发一个实时特征工程模块，将Open-Meteo原始数据转换为模型输入特征：

1. 数据映射要求：
   - temperature_2m → Dry_Bulb (直接使用)
   - dew_point_2m → Dew_Point (直接使用)
   - humidity_index = Dry_Bulb - Dew_Point (计算得到)
   - wind_speed 单位转换(m/s → mph)

2. 滞后特征生成：
   - load_lag_1h: 前1小时负荷
   - load_lag_24h: 前24小时负荷
   - load_lag_48h: 前48小时负荷  
   - load_lag_168h: 前168小时负荷
   - temp_lag_24h: 前24小时温度
   - temp_lag_168h: 前168小时温度

3. 滚动窗口特征：
   - load_rolling_mean_24h: 24小时平均负荷
   - load_rolling_std_24h: 24小时负荷标准差
   - load_rolling_mean_168h: 168小时平均负荷
   - temp_rolling_mean_24h: 24小时平均温度
   - temp_rolling_max_24h: 24小时最高温度
   - temp_rolling_min_24h: 24小时最低温度

4. 时间特征：
   - 小时、星期、月度的正弦/余弦编码
   - 周末、节假日布尔特征
   - 季节特征one-hot编码

5. 要求：
   - 与训练阶段的processed/step6_sequences.pkl特征完全一致
   - 支持增量计算(只处理新增数据)
   - 高效计算，适合实时处理

请实现FeatureGenerator类，并提供完整的特征转换流水线。
```

建议使用AI助手时按阶段进行，每个阶段完成后验证效果再进入下一阶段。

### 4. 持续集成
每个模块完成后，请AI助手生成相应的CI/CD配置，确保代码质量。
